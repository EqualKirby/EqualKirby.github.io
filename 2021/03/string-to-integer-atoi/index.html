String to Integer (atoi)

  2021-03-25 Word Count: 328 words Read Time: 1 minutes

水題，但還是有可以注意的點

題目連結

想法

這題目重點在越界檢查的部分和讀懂題目本身。

在上一篇有討論了怎麼檢查越界的這件事，這題與上一題不同的地方在於越界以後用該方向的極值做為表示。 譬如超過最大可以表達的正數就用最大值表示，低於最小可以表達的負數就用最小值表示。

我個人這題的實作就有點隨便，沒有特地把程式碼弄得精巧一點。 原本我想把 abs 替換成 * sign，但跟上一題有相同的狀況，如果 limit 是最小值然後 digit 是 $0$ 的狀況下就會爆開。

時間複雜度

顯而易見的 $\mathcal{O}(n)$

空間複雜度

顯而易見的 $\mathcal{O}(1)$

程式碼

class Solution {
public:
    int myAtoi(string s) {
        auto result = 0;
        auto sign = 1;
        auto limit = numeric_limits<int>::max();

        auto i = 0;
        while (i < s.size() && s[i] == ' ')
            i++;

        if (i < s.size() && (s[i] == '+' || s[i] == '-')) {
            if (s[i] == '-') {
                sign = -1;
                limit = numeric_limits<int>::min();
            }
            i++;
        }

        while (i < s.size() && isdigit(s[i])) {
            auto digit = (s[i] - '0') * sign;
            if (abs((limit - digit) / 10) < abs(result))
                return limit;

            result *= 10;
            result += digit;
            i++;
        }

        return result;
    }
};
