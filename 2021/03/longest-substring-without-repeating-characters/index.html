Longest Substring Without Repeating Characters

  2021-03-04   LeetCode Word Count: 848 words Read Time: 2 minutes

刷題第二篇

題目連結

想法

令輸入的字串為 $Str$，$S_i$ 是從 index $i$ 開始的最長且不重複的子字串
這樣的話，原先的問題就是求 $max(len(S_0), len(S_1), len(S_2),\dots, len(S_{len(Str)-1}))$

假設 $S_i$ 這個子字串就是 $Str[i, j)$，那麼只有可能是兩種情況：

  1. $j = len(Str)$
  2. $j \lt len(Str)$ 且 $\exists k \in [i, j), Str[k] = Str[j]$
    若否，會與 $S_i $定義矛盾

因此，我們就知道 $S_i$ 怎麼求了：針對每個 $S_i$ 都有個 index 從 $i$ 往後跑一次，檢查到如果有相同的字元就停止，即可得到 $j$
記錄有沒有相同的字元可以簡單地利用 table，因為題目有說可能出現的字元就是英文、數字和符號，那我就偷懶一點，假設他是 ASCII 囉

char 是不是 signed 並沒有被規定，更為嚴謹的方式是將其當 index 用之前要先轉到 unsigned

但其實再仔細觀察，會發現針對上面說的第二種情況，可以得到：

$max(len(S_{i+1}), len(S_{i+2}), len(S_{i+3}),\dots, len(S_k)) \le len(S_i)$

要證明也很簡單，上述不等式左側裡面的任一個子字串，其起始 index 皆比 $i$ 大，但右端點(半開區間)都是 index $j$
因此，其區間必然內嵌於 $[i, j)$ 裡面，長度自然不會超過 $len(S_i)$ (若否，會與 $S_i $定義矛盾)
那麼，我們就可以跳過 $S_{i+1}, S_{i+2}, S_{i+3},\dots, S_k$ 的計算，而是直接往下求 $S_{k+1}$

而且，index $j$ 也不需要往回設定從 $k+1$ 開始，因為上一次的 $S_i$ 保證了 $Str[i, j)$ 裡面沒有重複的字元
已知 $Str[k] = Str[j]$ 且 $k \in [i, j)$，$Str[k+1, j]$ 自然不會有重複的字元(不然會矛盾)

因此，這個問題實際上就可以變成所謂的 sliding window
利用兩個 pointer 維護子字串的起點 $start$ 與終點 $end$，並確保起點到終點之間是沒有重複的字元
在過程中持續監視子字串的長度，並且將最大值記錄下來，那就會是答案了

剩下的問題只剩下怎樣有效率地知道 $k$ 是多少？
我們可以將前面的 table 從記錄字元出現的有無，變成記錄看到 $end$ 為止，上一次該字元出現的 index
如果該 index 是大於等於 $start$，那就表示目前檢查的字元與 sliding window 內的字元有所重複

時間複雜度

可以觀察到 sliding window 有這幾個特性：

  • $start \le end$
  • $start$ 只會停留或是往右走
  • $end$ 只會停留或是往右走
  • 任何一次更新，$start$ 和 $end$ 之中一定至少有一個往右走

因此 $start$ 與 $end$ 最多就是各走 n 步，得到 $\mathcal{O}(n)$

空間複雜度

顯而易見的 $\mathcal{O}(1)$

程式碼

#include<algorithm>

class Solution {
public:
    int lengthOfLongestSubstring(string s) {
        using namespace std;
        
        int loc[256];
        fill(begin(loc), end(loc), -1);
        
        auto start = 0;
        auto len = 0;
        for(auto i = 0; i < s.size(); i++) {            
            if(loc[s[i]] >= start) {                
                start = loc[s[i]] + 1;
            } else {
                len = max(len, i - start + 1);
            }
            loc[s[i]] = i;
        }
            
        return len;
    }
};
